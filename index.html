Steps for git...
1. git init
2. git add .
3. git status
4. git commit -m "any message you want to give"
5. git branch -M main
6. git remote add origin https://github.com/Deepak7200/repoName.git
7. git push -u origin main
8. git pull // we use it to pull updated code from git(cloud) to local machine(vsCode)
9. git checkout -b setup // create new branch named "setup"
10. git checkout master // to move on other branch

Push an existing repository from the command line

2. git branch -M main
3. git push -u origin main

BEM Methodolgy:
1. https://getbem.com/
2. Used for creating a user variable in technical term
3. Use those names effectively in multiple sections (Reusable/Reusing the Variable Name)
i.e,
Let suppose, me as senior working on header with class name "buttons". And my junior working on content section with class "buttons".
So, to remove this conflict we use "BEM Approach" and give class name like
section1__buttons_body
section2__buttons_body

NOTE: It's a methodoly like camel case we can also use -> sec1ButtonsBody and sec2ButtonsBody
